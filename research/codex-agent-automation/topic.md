# Topic Contract: codex-agent-automation

## Goal

Track how Codex can be used as a local-first agent runtime for recurring
research, repo-backed memory, skills, MCP tools, hooks, and future SDK/server
orchestration.

## Scope

- Include: Codex automations, thread heartbeats, skills, MCP, hooks, Codex SDK,
  app-server, non-interactive execution, and practical local/VPS migration
  paths.
- Exclude: unrelated AI news, general model benchmark gossip, UI dashboard
  design, and production VPS implementation until the local MVP is stable.

## Priority

High

## Keywords

- Primary: Codex automations, Codex skills, Codex MCP, Codex SDK, Codex
  app-server, agent workflow automation
- Secondary: thread automation, heartbeat automation, codex exec, local-first
  agent, research automation, risk gate

## Source Policy

- Prefer official OpenAI Codex documentation and verified current-session Codex
  tool behavior.
- Use current web research when asking about recent product behavior, release
  changes, compatibility, or external tool updates.
- Record all source links used for material claims.
- Treat local callable tools as current for this machine, and document any
  mismatch with public docs.

## Report When

- Codex automation behavior, scheduling, permissions, or safety guidance
  changes.
- Skills, MCP, hooks, SDK, app-server, or non-interactive mode gain a relevant
  capability.
- A workflow improvement would reduce noise, improve source quality, or make
  topic threads safer.
- A risk appears around unattended writes, stale reports, duplicated findings,
  permissions, or cost.
